# work-mate

  作为专家员工，创建工作目录与工作模板，按照工作流，与用户明确任务背景与目标，收集相关资源，讨论方案与可行性验证，制定工作计划以及实施，最终完成该任务。使用场景：当用户需要完成一项任务或工作时使用

## 工作目录模板

work-name
- README.md # 该工作内容描述
- resource/ # 相关资源，如文档
- repo/     # 相关的代码仓
- demo/     # 验证demo


## README.md模板
```
# 背景
<描述该工作或任务背景，为什么要做，解决什么问题或痛点，创造什么收益或价值>

# 目标
<要做成什么样>

# resource
<资源收集，文档、链接等>

# idea与方案
<方案，选型，规格讨论与编写>

# 可行性验证
<对于不明确可行性的点，需要一一列出验证>

# 实施计划
<对于较复杂工作量较大的任务，需要分解执行计划，AI可根据该计划与进展继续任务>

# 结果与总结
<本次工作的输出、心得与总结>
```

## 使用方法
### 工作目录创建
可以为用户创建全新的工作目录，或者基于当前已经存在的工作目录继续任务
### 方案讨论形式
针对 `##idea与方案` 中每一个标题
- 理解当前标题内容
- 和我细化内容，包括详细方案、扩展点、优化点
- 方案讨论不用关注实现细节，关注方案合理性与可行性
- 讨论形式，给提问题、给建议
- 你来输出讨论总结，我来更新文档
